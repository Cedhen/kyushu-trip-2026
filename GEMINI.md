# Gemini Context: Trip-Template

## Project Overview
This repository contains a Progressive Web App (PWA) template for a family trip itinerary. The site is statically generated by fetching data from a connected Google Spreadsheet and injecting it into HTML, CSS, and JS templates.

## Architecture
- **Frontend**: Vanilla HTML, CSS, and JavaScript. No frontend frameworks (like React or Vue) or CSS frameworks (like Tailwind) are used.
- **Build System**: A Python script (`build_site.py`) acts as a static site generator. It uses the Google Sheets API (`google-api-python-client`) and `pandas` to read trip data, settings, and checklists, and replaces template placeholders (e.g., `{{SCHEDULE_CONTENT}}`) with the generated HTML.
- **PWA**: Supported via `manifest.json` and a Service Worker (`sw.js`).
- **Data Source**: A Google Spreadsheet configured via `DOC_ID` in the `.env` file. A Google Apps Script (GAS) URL is also used for form submissions (like the expense tracker).

## Directory Structure & Key Files
- `index.html`, `style.css`, `script.js`: **Source Template files**. These are the primary files you should modify. They contain placeholders like `{{WEATHER_CITY}}` and `{{PAGE_TITLE}}` that get replaced during the build.
- `build_site.py`: The core Python script that fetches data from Google Sheets, processes it, and generates the final static site.
- `public/`: The output directory for deployment. **DO NOT modify files inside this folder directly**, as they are strictly generated artifacts and will be overwritten every time the build script runs.
- `.env`: Contains necessary environment variables (`GOOGLE_API_KEY`, `DOC_ID`, `GAS_URL`).

## AI Assistant Guidelines & Development Workflow
When making modifications to this codebase, adhere to the following rules:

1. **Modify Templates, Not Output**: Always edit the source template files in the root directory. Never apply manual edits to the files inside the `public/` directory.
2. **Rebuild After Changes**: After making changes to the HTML/CSS/JS source files or the Python script, always recommend or run `python3 build_site.py` to regenerate the updated static files in the `public/` folder.
3. **Vanilla Stack Only**: Keep the frontend strictly Vanilla JS and CSS. Do not introduce dependencies like jQuery, React, or Tailwind unless explicitly requested by the user.
4. **Placeholder Management**: If introducing new dynamic features, you will need to update the data fetching logic within `build_site.py` and inject the new placeholders into `index.html` or `script.js`.
5. **PWA Updates**: If adding new static assets (e.g., icons, fonts, new HTML pages), make sure they are properly copied to the `public/` directory in `build_site.py` and consider adding them to the cache list in `sw.js`.
